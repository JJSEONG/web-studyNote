CSS의 기본적인 개념과 구조

⭐️ 사전적 정의

종속형 시트 또는 캐스케이딩 스타일 시트(Cascading Style Sheets, CSS)는 마크업 언어가 실제 표시되는 방법을 기술하는 스타일 언어(style sheet language)로, HTML과 XHTML에 주로 쓰이며, XML에서도 사용할 수 있다. W3C의 표준이며, 레이아웃과 스타일을 정의할 때의 자유도가 높다. 기본 파일명은 style.css이다. 마크업 언어(ex: HTML)가 웹사이트의 몸체를 담당한다면 CSS는 옷과 액세서리처럼 꾸미는 역할을 담당한다고 할 수 있다. 즉, HTML 구조는 그대로 두고 CSS 파일만 변경해도 전혀 다른 웹사이트처럼 꾸밀 수 있다. 현재 개발 중인 CSS3의 경우 그림자 효과, 그라데이션, 변형 등 그래픽 편집 프로그램으로 제작한 이미지를 대체할 수 있는 기능이 추가되었다. 또한 다양한 애니메이션 기능이 추가되어 어도비 플래시를 어느 정도 대체하고 있다.

출처 : https://ko.wikipedia.org/wiki/CSS

Web의 스타일을 꾸며주는 역할을 도맡아서 진행하는 언어가 CSS(Cascading Style Sheet)이다. 이 언어는 HTML언어로 마크업이 된 상태의 문서를 가져와 문서에 스타일을 주고, 전혀 다른 웹사이트처럼 변경할 수 있다. 또한 다양한 애니매이션 기능 등을 사용하면 더욱 더 멋지고 아름다운 홈페이지를 만들 수 있다.

⭐️ HTML의 구조

HTML은 마크업 언어로 태그를 이용하여 WebSite를 구성한다.

Html html 태그를 사용하여 문서의 처음과 끝 즉, 문서의 범위를 나타낸다.
Head head 태그는 제목이나 검색키워드 등 문서의 정보를 가지는 태그이다.
body body 태그 안에 여러 태그들을 이용하여, 웹문서를 만들 때 body 태그 안에 작성한다.(문서의 구조)
<html> [html의 여는태그]
<head> [head의 여는태그]
<body> [body의 여는태그]
</body> [body의 닫는태그]
</head> [head의 닫는태그]
</html> [html의 닫는태그]

위의 내용과 같이 HTML의 기본적인 구조는 <여는태그>내용<닫는태그>로 구성이 되어진다.

⭐️ HTML의 부모-자식 구조

HTML의 구조는 부모-자식 간의 구조로 이루어진다. 부모-자식 간의 구조를 잘 이해하여야, 추후에 CSS에서 스타일을 줄 때 어느 특정 요소에 스타일을 주기가 쉽다. 부모와 자식은 상대적으로 이루어지며, 부모요소 아래의 자식요소가 또 다른 자식요소를 가질 수 있다.

<section>
<h1> 색의종류 </h1>
<ul>
<li> RED </li>
<li> GREEN </li>
<li> BLUE </li>
</ul>
</section>

위의 예시에서 부모-자식 요소를 알아보자.

<section> 태그 : <h1> 태그와 <ul> 태그의 부모

<h1> 태그 : <section> 태그의 자식

<ul> 태그 : <section> 태그의 자식이자 <li> 태그의 부모

<li> 태그 : <ul> 태그의 자식

위의 내용과 같이 HTML의 부모-자식 구조는 태그간 상대적으로 이루어진다.
