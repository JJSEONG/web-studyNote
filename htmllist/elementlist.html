HTML의 요소(Element)의 종류

⭐️ 요소(Element)의 종류

HTML의 요소(Element)는 수없이 많은 종류가 있습니다. 그 중에서도 HTML 문서에 주로 사용되며, 개인적으로 Web 개발을 할 때, 이정도는 최소한으로 알아야 한다고 생각하는 것 들을 정리하였기 때문에 부족할 수 있습니다. 더욱 더 자세한 내용을 알고 싶으시면, MDN 문서 등 공식적인 문서를 찾아보는 것도 좋은 방법입니다.

⭐️ !DOCTYPE 선언문

태그(Tag)를 선언하기 전에, <!DOCTYPE html> 을 선언하여 이 웹페이지가 HTML로 구성이 된다는 점을 웹페이지에 알려주는 역할입니다. 이 선언이 없이 HTML의 문서를 작성하면, 웹브라우저에서 표준 모드(Standards Mode)가 아닌 비표준 모드(Quirks Mode) 로 인식해 크로스 브라우징에 오류가 발생 할 수 있어, 개발환경 외의 다른 브라우저에서는 아무리 깔끔하고 멋있게 작성한 문서라도 원하지 않는 모습으로 표현될 수도 있다.

⭐️ <html> 태그

<html> 태그는 !DOCTYPE을 선언 한 후 html의 문서의 시작과 끝을 알려주는 태그이다.
<html> 태그의 속성으로는 lang 이라는 속성이 들어가며, 이는 language의 약자로, 웹브라우저의 표준 언어를 지정하는데 사용된다.
en = 영어 / ko = 한국어 / ja = 일본 등의 속성값이 존재한다.

⭐️ <head> 태그

<head> 태그는 <html> 태그 밑에 작성되며, 웹브라우저의 정보 이름이나 언어방식 등이 작성되어있다.
대표적으로 <meta> 태그와 <title> 태그가 있다.

속성명 의미
<meta> 태그는 웹브라우저의 언어, 정보, 키워드 등을 작성할 수 있다.
<title> 웹브라우저의 제목을 작성할 수 있다.

⭐️ <link> 태그

빈 요소(Empty Element)이며, head 태그 안에 사용된다. 외부에서 문서를 가져올 때 사용하는 태그이다. 주로, css, javascript 같은 문서를 불러오는데 사용한다.

<link rel = 현재 문서와의 관계 href = 외부 문서의 링크(경로)>

rel 속성
: 불러 올 외부 문서와 현재 작성되고 있는 문서와의 관계를 나타낸다. 주로 사용되는 stylesheet 는 css 문서를 불러올 때 사용되는 속성값이고, icon은 favicon을 불러올 때 사용되는 속성값이다.
href 속성
: 불러 올 외부 문서의 링크(경로)를 작성하여 현재 작성되는 페이지에서 작성된 속성값의 경로에서 문서를 가져오는 역할을 한다.

⭐️ <body> 태그
Block Level Element

<body> 태그는 html 문서의 내용을 작성하며, 여러가지 요소들을 구조와 문법에 맞게 작성하면, 웹페이지를 원하는 구조로 작성 할 수 있다.

<!DOCTYPE html>
<html lang="ko">
<head>
<meta charset="UTF-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>Document</title>
</head>
<body>
Contents
</body>
</html>

위의 예시와 같이 DOCTYPE 선언 후 html태그가 나오고, head태그와 body태그가 차례로 나오는 구조를 가지게된다.

⭐️ Heading 태그
Block Level Element

제목을 단계별로 나타내며, h1~h6으로 구성되어있다. 문서의 대제목부터 차례로 사용하며, 대제목은 h1 태그를 사용하되, h1 태그는 중복 사용은 피하는 것이 좋다.

<h1> 대제목 </h1>
<h2> 소제목 </h2>
<h3> 소소제목 </h3>
등 제목간의 중요도를 나타낼 때 사용한다.

⭐️ <p> (Paragraphs) 태그
Block Level Element

하나의 문장이나 문단을 설정할 때 사용하는 태그이다.

<p> 안녕하세요? </p>
<p> 정종성입니다. </p>

⭐️ <hr /> 태그
Block Level Element

빈 요소(Empty Element)로 단독으로 사용되며, 주제가 변경되거나 구분을 주고 싶을 때 사용되며, 사용시 아래 예시와 같이 구분선이 생성된다.

<span> 첫번째주제 </span> <hr />
<span> 두번째주제 </span>

⭐️ <blockquote> 태그
Block Level Element

일반적으로 어떠한 문구를 인용하여 가져올 때 사용된다. 통상적으로 긴 인용문을 가져올 때 사용하는 태그이다.
속성으로는 cite 속성이 있으며, cite 속성값으로는 인용된 정보의 출처(URL)을 입력한다.

<blockquote cite = "인용문의 출처(URL)"> 긴 인용문 </blockquote>

⭐️ <q> 태그
Inline Level Element

<q>태그는 짧은 인용문을 설정할때 사용한다.
<blockquote>는 긴 인용문을 설정할때 사용하고, 블록 레벨요소인 점이 다르다.
속성으로는 <blockquote>와 동일하게 cite 속성이 있으며, cite 속성값으로는 인용된 정보의 출처(URL)을 입력한다.

<q cite = "인용문의 출처(URL)"> 짧은 인용문 </q>

용어의 대한 설명을 지정하여 작성하여준다. 하나의 용어에 여러개의 설명이 있다면, <dd> 태그를 추가하여 작성해주면 된다.

⭐️ <pre> 태그
Block Level Element

서식이 미리 지정된 텍스트를 보여주는 태그이다. html 문서 내의 줄바꿈이나 공백을 특정한 태그와 함께 사용하여 표현해야 하는데, pre 태그 안에서는 작성한 내용대로 화면에 표시한다.

<pre>
pre 태그 안에서 <br>태그가 없이
줄바꿈을 진행해도
줄바꿈이 표현이 된다.
</pre>

⭐️ <br /> 태그
Inline Level Element

빈 요소(Empty Element)로 단독으로 사용되며, 줄바꿈을 하고 싶을 때 사용한다.

<span> 안녕하세요? </span> <br />
<span> 정종성입니다. </span>

⭐️ <ol>,<ul> 태그
Block Level Element

이 두개의 태그는 목록을 나타낼 때 사용한다. 순차적 목록(Ordered list)인 <ol>, 비순차적 목록(Unordered list)인 <ul> 이 있다.
이 두개의 태그 안에는 <li> 태그만이 자식으로 들어갈 수 있다.

<ol>
<li> 첫번째 </li>
<li> 두번째 </li>
<li> 세번째 </li>
</ol>

  2. 두번째
  3. 세번째

<ol> 태그는 순차적인 목록을 작성할 때 사용한다.

<ol type= "a" start = "4" reversed>
<li> 첫번째 </li>
<li> 두번째 </li>
<li> 세번째 </li>
</ol>

위와 같이 작성을 하게되면 아래와 같은 결과를 가지게된다.

  1. 첫번째
  2. 두번째
  3. 세번째

<ol> 태그의 속성인 type 속성에 속성값으로 " 1(기본값), a, A, i, I "가 있으며, 항목에 사용할 번호의 종류 설정한다.
<ol> 태그의 속성인 start 속성에 숫자를 입력하면 항목의 시작 숫자 설정한다.
<ol> 태그의 속성인 reversed는 말 그대로, 순서를 역순으로 설정해서 보여준다.

<ul>

<ul> 태그는 비순차적인 목록을 작성할 때 사용한다.

<ul type= "square">
<li> 첫번째 </li>
<li> 두번째 </li>

  • 세번째

<ul> 태그의 속성인 type 속성에 속성값으로 " disc(기본값), square, circle, none " 이 있으며, 항목에 사용할 기호의 종류 설정한다.

⭐️ <li> 태그
list-item Element

이 태그는 목록의 리스트를 나타낼 때 사용한다. <li> 태그는 단독으로 사용이 불가능하며, 꼭 <ul> 혹은 <ul> 태그를 부모로 두어야 사용이 가능하다.
위의 <ol>, <ul> 의 예시만 보아도 사용법을 알 수 있다.

⭐️ <dl>,<dt>,<dd> 태그
Block Level Element

위의 태그들은 용어에 대한 설명을 지정하여 사용하는 태그이다. <dl> 태그 안에는 <dt>, <dd> 외의 태그는 자식으로 존재가 불가능하다.

  • <dl> : 해당 범위가 용어와 용어에 대한 설명임을 정의한다.

  • <dt> : 용어를 지칭한다.

  • <dd> : 용어에 대한 설명을 일컫는다.

<dl>
<dt> Firefox </dt>
<dd> 모질라 재단에서 만든 웹 브라우저 </dd>
</dl>

위와 같이 작성을 하게되면 아래와 같은 결과를 가지게된다.

모질라 재단에서 만든 웹 브라우저

용어의 대한 설명을 지정하여 작성하여준다. 하나의 용어에 여러개의 설명이 있다면, <dd> 태그를 추가하여 작성해주면 된다.

⭐️ <div> 태그
Block Level Element

<div> 태그는 Division의 약자로 문서 내에서 영역이나 섹션을 정의하기 위한 태그이다. 아무 의미를 가지지않는 태그이며, 컨텐츠를 채워넣고 스타일을 위한 태그로 많이 사용된다.

⭐️ <span> 태그
Inline Level Element

<span> 태그는 <span> 태그와 같이 아무 의미를 가지지않는 태그이며, 용도도 비슷하지만 인라인 레벨요소로 구분해서 사용해야하며, 주로 인라인 요소들을 묶을 때 사용한다.

⭐️ <a> (anchor) 태그
Inline Level Element

<a> 태그는 하이퍼링크(Hyperlink)를 설정해주는 태그이다. 주로, 페이지 내에서 다른 페이지로 이동시켜주거나, 같은 페이지의 특정 페이지로 이동, 이메일이나 전화번호 등도 연결할 수 있는 하이퍼링크(Hyperlink)를 설정한다. <a> 태그에 같이 사용되는 속성을 알아보자.

속성 의미
href herf 속성값으로는 이동할 링크(URL)을 작성한다.
target 링크(URL)을 클릭 시, 브라우저의 위치를 설정한다.
<a href = "이동할 URL 주소(경로)" target = "_self (기본값) / _blank"> 이동할 경로의 설명 </a>

target의 속성값에는 _self 와 _blank 가 있다.

_self : 현재 페이지(현재 탭)에서 링크로 이동
_blank : 새 페이지(새 탭)를 열어 링크로 이동

<a href = "https://www.naver.com" target = "_self"> 여기서 네이버로 이동 </a>
<a href = "https://www.naver.com" target = "_blank"> 새창에서 네이버로 이동 </a>

위와 같이 사용되는 편이 많으며, 그 외의 사용법도 알아보자.

<a href = "#"> 임시링크 </a>
href 속성에 "#" 의 속성값을 넣으면 임시링크를 지정해 두는 것으로, 추후에 서브페이지를 만들거나, 이동 할 주소를 알았을 때, "#"을 삭제하고 링크를 적어 넣어주면 된다.
임시링크

<a href = "mailto:oooo@ooo.com"> 메일로 연결 </a>
href 속성에 "mailto:메일주소" 의 속성값을 넣으면 아웃룩 계정으로 메일을 작성 할 수 있다.
아웃룩 메일로 연결

<a href = "tel:000-0000"> 전화연결 </a>
href 속성에 "tel:전화번호" 의 속성값을 넣으면 웹에서는 반응하지 않지만, 모바일에서는 전화창으로 넘어가게되며, 지정된 전화번호가 자동으로 작성된다.
전화창으로 연결

<a href = "#id명"> #id명으로 이동 </a>
href 속성에 "#id명" 의 속성값을 넣으면 해당 id값을 가진 부분으로 이동하게 된다.
a태그 설명으로 이동!

⭐️ <b>,<strong> 태그
Inline Level Element

두 태그 모두 기본적으로 텍스트를 볼드체로 변경시켜준다. 그러나 그 둘은 서로 다른 의미로 사용이 된다.

  • <b> : 단순히 다른 텍스트와 다르게 구별하고 싶을 때 사용

  • <strong> : 중요한 의미를 갖는 텍스트를 나타낼 때 사용 (강조, 경고, 주의사항 등)

<p> <b> HTML </b> 은 Hyper Text Markup Language의 약자이다. </p>
<p> 자전거를 타기 전에. <strong> 헬멧을 꼭 착용 </strong> 하세요. </p>

⭐️ <i>,<em> 태그
Inline Level Element

두 태그 모두 기본적으로 텍스트를 이탤릭체로 변경시켜준다. 그러나 그 둘은 서로 다른 의미로 사용이 된다.

  • <i> : 기술용어, 외국어, 등장인물의 생각 등 일반적인 글자와 구분하기 위해 사용한다.

  • <em> : 강조하고 싶은 텍스트를 나타낼 때 사용하며, 중첩이 가능해 많이 사용할수록 강조의 의미가 강해진다.

<p> 라틴어 문구 <i> veni, vidi, viciL </i> 는 음악과 예술, 문학에 자주 등장합니다. </p>
<p> <em><em> I Love You </em></em> </p>

⭐️ <u>,<ins> 태그
Inline Level Element

두 태그 모두 밑줄을 표현해준다. 그러나 그 둘은 서로 다른 의미로 사용이 된다.

  • <u> : 맞춤법이 틀린 텍스트를 표현할 때 사용한다.

  • <ins> : 없던 내용이 추가되어 변경되었다는 것을 나타낼 때 사용한다.

<p> 맞춤법이 틀리면 <u> 않돼요. </u> </p>
<p> 맞춤법이 틀리면 <u> 않돼요. </u> <ins> 맞춤법 확인합시다! </ins> </p>

위와 같이 작성하면 아래와 같은 결과를 가지게된다.

맞춤법이 틀리면 않돼요.

맞춤법이 틀리면 않돼요. 맞춤법 확인합시다!

⭐️ <s>,<del> 태그
Inline Level Element

두 태그 모두 텍스트 중앙 취소선을 표시해준다. 그러나 그 둘은 서로 다른 의미로 사용이 된다.

  • <s> : 정확하지 않고, 관련성이 없는 내용을 나타낼 때 사용한다.

  • <del> : 기존에 있던 내용이 삭제되고 변경되었다는 것을 나타낼 때 사용한다.

<p> 감자 1kg에 <s> 10,000원 </s> <em> 할인가 7,900원 </em> </p>
<p> 내가 좋아하는 색은 <del> 핑크색 </del> <ins> 보라색 </ins> </p>

위와 같이 작성하면 아래와 같은 결과를 가지게된다.

감자 1kg에 10,000원 할인가 7,900원

내가 좋아하는 색은 핑크색 보라색 이다.

⭐️ <sup>,<sub> 태그
Inline Level Element

이 태그들은 위 첨자, 아래 첨자를 표현해준다.

  • <sup> : Superscripted text의 약자로 위 첨자를 나타낸다.

  • <sub> : Subscript text의 약자로 아래 첨자를 나타낸다.

<p> 2 <sup> 2 <sup> </p>
<p> CO <sub> 2 <sub> </p>

위와 같이 작성하면 아래와 같은 결과를 가지게된다.

22

CO2

⭐️ <small> 태그
Inline Level Element

<small> 태그는 작은 텍스트로 표현되며, 주로 저작권을 표시할 때 사용한다.

<p> 기본글자 <small> 저작권표시 <small> </p>

위와 같이 작성하면 아래와 같은 결과를 가지게된다.

기본글자 저작권표시

⭐️ <ruby>, <rt>, <rb>, <rp> 태그
Inline Level Element

<ruby> 태그는 동아시아 글자를 표현할 때 주로 사용하는 루비문자이다. 주로 일본어나 중국어(한자) 등 위에 발음을 작은 텍스트의 위/아래 첨자로 알려주는 루비주석을 설정할 때 사용한다.

  • <ruby> : 루비문자의 범위를 지정합니다.

  • <rt> : Ruby Top의 약자로 루비문자의 상단 주석을 설정한다.

  • <rb> : Ruby Bottom의 약자로 루비문자의 하단 주석을 설정한다.

  • <rp> : Ruby Parentheses의 약자로 루비문자를 지원하지 않는 브라우저에 괄호를 표시해준다.

<ruby>
漢 <rp> ( </rp> <rt> kan <rt> <rp> ) </rp> <sup>
字 <rp> ( </rp> <rt> ji <rt> <rp> ) </rp> <sup>
</ruby>

위와 같이 작성하면 아래와 같은 결과를 가지게된다.

漢( kan ) 字( ji )
